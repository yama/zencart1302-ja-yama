v1.3.0.2-l10n-jp-6から1.3.0.2-l10n-jp-7へのアップデート方法

v1.3.0.2-l10n-jp-6からアップデートするにはzen-cart-1.3.0.2-l10n-jp-6-to-v1.3.0.2-l10n-jp-7-patch.zipをダウンロードし以下の手順で進めてください

1.各ファイルをサーバにアップロードしてください。
mysql_upgrade_zencart_1302jp6_to_1302jp7.sql はアップロードする必要ありません
-----------------------------------------------
次のファイルはZen Cartオリジナルファイルを修正しています。 もし、あなたが1.3.0.2-l10n-jp-6のZen Cartを使用しており、 これらのファイルを独自にカスタマイズしていなければ そのまま上書きしてください。

もし、そうでなければ、WinMergeを使用して、 あなたが運用しているZen Cart上のこれらのファイルとの 修正差分を確認し、手動で差分をマージしてください。

v1.3.0.2-l10n-jp-6からの修正点

v1.3.0.2-l10n-jp-7で修正された内容:

  • PHP 5.3に対応した
  • PayPalがExpressCheckout, WebPaymentPlus対応になった。
  • クーポン設定で挙動がおかしくなるバグ
    クーポンを設定すると日本語と英語で設定していても、どちらかに統一されてしまう
  • リンク商品のカテゴリ名とイメージが勝手に変わってしまう
    商品のリンクを作成するとカテゴリIDの大きなカテゴリ名とイメージが表示されてしまいます。
  • 管理画面のindex.phpのバグについて
    管理画面でのindexページで、新しい顧客の一覧が表示されていますが、クリックしても名前や、ブラウザよって文字化けをおこしたりして、全く機能していない。
  • ポン券の商品別の制限でエラー
    管理画面 > ギフト券・クーポン券 > クーポン券の管理でクーポンを登録して、制限を押しますと、商品別の利用制限のところで、Catchable fatal error: Object of class queryFactoryResult could not be converted to string in /htdocs/admin/includes/functions/html_output.php on line 333 のような形で、エラーが出ております。
  • デモデータの sql ファイルにバグがあります
    products テーブルのレコードが products_id = 229 までなのに対して、products_description テーブルのレコードは products_id = 234 まで使われています。
  • ウェルカムギフト券について
    ギフト券もクーポン券も9999円の設定になっており10000円以上に変更ができない
  • 管理画面からキーワードを埋め込む時のエラー
    管理画面の商品・カテゴリの管理から、カテゴリにキーワードを埋め込むときに、1062 Duplicate entry '1219-1' for key 1のように表示される場合がある。
  • バナーの管理の開始日，終了日のバグ
    バナーを登録時に開始日，終了日を設定すると2036/11/30などのありえない日付になってしまいます。
  • Re: Zen-cart Pro
    一般設定→全商品の設定→タイトルの上から4番目 Key: PRODUCT_ALL_LIST_NAME を「商品価格の表示」これ違います、正解は「商品名の表示」です
  • salemaker.phpのバグ？
    本来なら全体を通して1度しか行わないはずの重目の処理が、セール終了日だけ、アクセス毎に実行され、サーバーが重くなる不具合の対応
  • 数量訂正時のバグ？
    カートの内容にてこのダウンロード商品の数量は訂正できません。
  • 買い物を確定した際、クロネコ＠ペイメントのボタンが表示されない
  • Zen Cart v1.3.0.2-l10n-jp-3のバグ報告
    顧客のデフォルトアドレスがZONEテーブルに存在した場合、住所変更時にZONEテーブルに存在しない国を選ぶことが出来なくなる。
    デモ商品をインストールした時の症状ですが、ベーシック認証が掛かります。
  • プロジェクト / v1.3.0日本語化 / バグトラック / 36
    ョッピング中の「送料の計算」で更新を２回めにクリックすると、Cannot redeclare class xxxxx
  • プロジェクト / v1.3.0日本語化 / バグトラック / 35
    ヤマト運輸を選ぶと「佐川急便の配送を有効にする」と表示される
  • ダブルクリックによる二重注文は防止できるか？
  • tpl_modules_create_account.phpで発見！これって間違いじゃないですか？？
    アカウント作成の画面に表示される「都道府県」なのですが、これだけinputにidがありません。
  • カート内でオプション付の商品を区別する為のユニークコードを生成する箇所のバグ。
    break が有効になっているとオプションでチェックボックスを使用したときにそれ以降のオプションの値がユニークコードの生成に使用されずに同一のものと見なされてしまうことがある。
